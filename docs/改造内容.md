# true-mem 记忆体系改造内容

> 基于 Agent-Skills-for-Context-Engineering 项目中 memory-systems 和 filesystem-context 两个技能的架构指导，对 true-mem（v1.4.1）进行四阶段系统化改造。

---

## 一、改造范围总览

```
Phase 0: 全局安装 memory-systems + filesystem-context 两个 skill
Phase 1: 配置优化 + 行为约定写入 AGENTS.md
Phase 2: 时序过滤 + 自动清理脚本
Phase 3: Plan Persistence + Scratch Pad 模式落地
Phase 4: 记忆质量评估 + true-mem → ontology 实体关联桥接
```

## 二、Phase 0：全局安装 Context Engineering Skills

**操作：** 将 `Agent-Skills-for-Context-Engineering` 项目中的两个 skill 安装到 OpenCode 全局技能库。

```powershell
Copy-Item "D:\Program Files\Agent-Skills-for-Context-Engineering-main\skills\memory-systems\*" `
  "$env:USERPROFILE\.config\opencode\skills\memory-systems\" -Recurse -Force

Copy-Item "D:\Program Files\Agent-Skills-for-Context-Engineering-main\skills\filesystem-context\*" `
  "$env:USERPROFILE\.config\opencode\skills\filesystem-context\" -Recurse -Force
```

**作用：** 安装后 OpenCode 在对应语境下自动加载技能内容，提供架构指导。

## 三、Phase 1：配置优化 + 行为约定

### 3.1 true-mem 配置优化

**文件：** `~/.true-mem/config.jsonc`

| 配置项 | 旧值 | 新值 | 影响 |
|--------|------|------|------|
| `storageLocation` | `"legacy"` | `"opencode"` | 数据迁移到 OpenCode 统一目录 |
| `injectionMode` | `1` (ALWAYS) | `0` (SESSION_START) | 每会话节省 ~76% token |
| `maxMemories` | `20` | `25` | 补偿 SESSION_START 模式的信息密度 |

### 3.2 AGENTS.md 追加 §7：Memory Discipline Rules

**文件：** `~/.config/opencode/AGENTS.md`（末尾追加）

新增 4 条强制行为规则：

| 规则 | 来源 | 作用 |
|------|------|------|
| **§7.1 Scratch Pad** | filesystem-context 模式1 | 工具输出 >2000 token 写入文件，仅返回摘要 |
| **§7.2 Plan Persistence** | filesystem-context 模式2 | 跨 3 轮以上任务创建 YAML 计划，每轮重读 |
| **§7.3 Memory Validity Check** | memory-systems Gotcha 7 | true-mem 注入记忆与代码库一致性检查 |
| **§7.4 Session Checklist** | memory-systems 框架 | 新会话启动时自动检查 plan/memory/scratch |

## 四、Phase 2：时序过滤 + 自动清理

### 4.1 时序有效性过滤

**文件：** `scripts/memory-temporal-filter.py`

**问题：** true-mem 记忆一旦存储即永久有效，没有 `valid_from/valid_until` 机制。当事实变化时（如 Python 3.12→3.13），旧记忆会变成"上下文毒药"。

**方案：** 在 true-mem 外部加一层时序标记机制，通过 `temporal_index.jsonl` 标记记忆的替代关系。

**API：**
```bash
# 标记旧记忆被替代
echo '{"memory_id":"mem_new","subject":"Python版本","valid_from":"2026-04-15","supersedes":"mem_old"}' \
  >> ~/.config/opencode/true-mem/temporal_index.jsonl

# 过滤被替代的记忆
py scripts/memory-temporal-filter.py
```

### 4.2 Scratch 文件自动清理

**文件：** `scripts/scratch-cleanup.ps1`

**功能：** 清理 7 天前的 scratch 文件，保留最近 50 个。支持 `-DryRun` 预览模式。

```powershell
# 预览
.\scripts\scratch-cleanup.ps1 -DryRun
# 执行
.\scripts\scratch-cleanup.ps1
```

## 六、Phase 4：记忆质量评估 + 实体关联

### 6.1 记忆质量评估探针

**文件：** `scripts/memory-eval.py`

**四个评估维度：**

| 维度 | 检测内容 | 评分方式 |
|------|---------|---------|
| `noise` | 10 种噪声模式（system-reminder、XML 标签、AI meta-talk 等） | 1-10 分 |
| `duplicate` | 基于 SHA256 的归一化 summary 去重 | 重复计数 |
| `classification` | constraint 被判为 noise、preference 含 AI meta-talk | 合理性检查 |
| `temporal` | temporal_index.jsonl 交叉验证 | 替代关系 |

**使用方法：**
```bash
# 诊断所有维度
py scripts/memory-eval.py

# 定向检查 + 自动修复
py scripts/memory-eval.py --dimension noise --fix
py scripts/memory-eval.py --dimension duplicate --fix
```

**首次运行结果：**
- 噪声：检测到 4 条（1 system-reminder + 3 procedural XML 噪声），已自动删除
- 重复：3 条 procedural（同一条 SDD 规则存 3 次），在噪声维度已被删除

### 6.2 true-mem → ontology 实体关联桥接

**文件：** `scripts/memory-to-ontology-bridge.py`

**功能：** 从 true-mem 记忆自动提取结构化实体和关系，写入 ontology 知识图谱。

**提取规则：**

| true-mem 分类 | 映射 entity 类型 | 关系 |
|--------------|----------------|------|
| preference / constraint | Decision | Person --decided--> Decision |
| preference（技术偏好） | Language / Technology / Framework / Tool | Person --prefers--> Technology |
| project-scope 记忆 | Project | Project --has_rule--> Decision |

**使用方法：**
```bash
# 预览（不写入）
py scripts/memory-to-ontology-bridge.py

# 写入 ontology
py scripts/memory-to-ontology-bridge.py --apply
```

**首次运行结果：**
- 2 条总记忆 → 1 条清洁（过滤 1 条 AI meta-talk）
- 创建实体：Project(amazon-ad)、Decision(search-mode constraint)、Person(SYZ)
- 创建关系：`SYZ --decided--> search-mode`、`amazon-ad --has_rule--> search-mode`

### 6.3 本体结构

**文件：** `memory/ontology/schema.yaml`

```yaml
types:
  Person, Project, Decision, Language, Technology, Framework, Tool

relations:
  prefers, constrains, has_rule, decided, uses
```

**文件：** `memory/ontology/graph.jsonl`（JSONL append-only）

## 八、关键架构决策

1. **所有脚本在 true-mem 外部运行**，不修改 true-mem 核心插件代码（npm 包），确保升级兼容
2. **时序标记采用 JSONL append-only** 格式，与 ontology 存储模式一致
3. **记忆评估不删数据，只标记**——`--fix` 将记忆标记为 deleted（而非物理删除），可恢复
4. **bridge 为单向管道**：true-mem → ontology，不反向写入，避免记忆源污染
