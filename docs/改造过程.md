**目标：** 将注入策略从 ALWAYS 改为 SESSION_START，节省 76% token；迁移存储到 OpenCode 统一目录。

**步骤：**

  "injectionMode": 0,                // 1 → 0 (SESSION_START)

### 2.3 session-check.py 替换方案

**踩坑：** Windows 系统未安装 sqlite3 CLI，原计划的 `session-init.ps1` 依赖 sqlite3 查询无法执行。

**解决：** 用 Python 重写为 `session-check.py`，直接调用 Python sqlite3 模块读取 DB。

**功能：**
- 检测 DB 位置（`opencode` 或 `legacy`）
- 按分类统计活跃记忆数量
- 读取 active plans
- 报告 scratch 文件摘要

## 三、Phase 2：时序过滤 + 自动清理

### 3.1 memory-temporal-filter.py

**实现逻辑：**

```
1. 检测 DB 位置：先查 opencode 目录，fallback 到 legacy
2. 读取 temporal_index.jsonl（JSONL 格式，每行一条时序标记）
3. 查询 true-mem DB 所有活跃记忆（按 strength DESC）
4. 收集所有被 supersedes 标记的记忆 ID
5. 过滤：排除被替代的旧记忆
6. 输出 JSON + stderr 报告
```

**时序标记格式：**
```jsonl
{"memory_id":"mem_abc123","subject":"Python版本","valid_from":"2026-01-01"}
{"memory_id":"mem_def456","subject":"Python版本","valid_from":"2026-04-15","supersedes":"mem_abc123"}
```

**踩坑与解决：**
- `python3` 别名不存在 → 改用 `py` 命令
- DB 已在 Phase 1 配置为 opencode 目录，但尚未迁移数据 → 手动 `Copy-Item` 完成迁移

### 3.2 scratch-cleanup.ps1

**踩坑：** PowerShell 5.1 对 UTF-8 无 BOM 文件处理不稳定（中文注释乱码）。

**解决：** 使用 `Out-File -Encoding UTF8` 写入，确保 BOM 头正确。

**逻辑：**
```powershell
1. 列出 scratch 目录所有文件
2. 保留：最近 50 个文件 OR 7 天内修改的文件
3. 删除：同时不满足两个条件的文件
4. -DryRun 参数仅预览，不执行删除
```

## 五、Phase 4：评估 + 实体关联

### 5.1 memory-eval.py

**四个评估维度实现细节：**

| 维度 | 实现方式 | 阈值 |
|------|---------|------|
| noise | 10 种正则模式匹配（system-reminder, XML 标签, AI meta-talk 等） | score ≥ 7 视为噪声 |
| duplicate | SHA256 hash of `normalize(summary)` | 多条相同 hash 视为重复 |
| classification | 规则检查（constraint 不应含问句，preference 不应含 AI 元对话） | 硬规则判定 |
| temporal | 加载 temporal_index.jsonl，检查 superseded 关系 | — |

**首次运行数据：**
```
Active memories: 6
Noise detected: 4 (1 system-reminder score=10, 3 procedural XML score=9)
Duplicates: 2 sets (same SDD rule ×3)
Classification issues: 5
```

**--fix 执行结果：**
```
--dimension noise:  4 deleted (noise)
--dimension duplicate: PASS (3 already in noise batch)
Remaining active: 2
```

### 5.2 memory-to-ontology-bridge.py

**实体提取逻辑：**

```
1. 查询 true-mem 活跃记忆
2. 按 project_scope 分组 → 创建 Project 实体
3. 提取 preference/constraint/procedural → 创建 Decision 实体
4. 固定创建 Person(SYZ) 实体
5. 关键词匹配提取 Technology/Language/Framework/Tool
6. --apply: 写入 graph.jsonl + 初始化 schema.yaml
```

**噪声过滤：**
- 硬编码过滤列表：`<system-reminder>`, `OMO_INTERNAL`, `<skill-instruction>`, `<true_memory_context>`
- AI meta-talk 检测：以 "Agent " 开头的 preference 不映射到实体

**首次运行数据：**
```
Total memories: 2 → Clean: 1 → Entities: 3, Relations: 2
```

### 5.3 DB 迁移

**踩坑：** true-mem 的自动迁移功能仅在**完全重启 OpenCode 进程**时触发，`--continue` 模式不会触发。

**手动迁移：**
```powershell
Copy-Item "$env:USERPROFILE\.true-mem\memory.db" "$env:USERPROFILE\.config\opencode\true-mem\memory.db"
```

## 七、踩坑总结

| # | 问题 | 根因 | 解决方案 |
|---|------|------|---------|
| 1 | sqlite3 CLI 不可用 | Windows 默认无 sqlite3 | Python 重写查询逻辑 |
| 2 | DB 自动迁移未触发 | `--continue` 不触发生命周期 hook | 手动 Copy-Item |
| 3 | python3 别名不存在 | Windows py launcher | 统一使用 `py` 命令 |
| 4 | PS1 中文乱码 | UTF-8 无 BOM 兼容性问题 | `Out-File -Encoding UTF8` |
| 5 | 记忆含 XML 噪声 | true-mem 误提取系统内部标签 | memory-eval.py 噪声检测+清理 |
| 6 | 同一记忆重复存储 3 次 | true-mem 内容去重依赖 exact hash，非语义 | memory-eval.py 重复检测 |
