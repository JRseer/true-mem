# trueMem 记忆浏览器 (Viewer) V1.0

trueMem Viewer 是为 trueMem 持久化记忆插件量身定制的本地 Web 可视化管理控制台。
无需手写 SQL 或翻看日志，直接在浏览器中即可安全、高效地管理你的 AI 记忆库。

## ✨ 核心功能

### 1. 记忆列表 (Feed)
- **多维过滤**：按存储类型（短期/长期）、记忆分类（偏好/约束/决策等）、状态（活跃/已衰减）快速筛选。
- **全文检索**：支持对记忆摘要进行搜索。
- **详情查看**：展开单条记忆，查看其来源任务、版本号，以及重要性、新颖性、置信度等内部评分指标。
- **快捷操作**：支持对错误记忆进行软删除/恢复，或手动修正其记忆分类。

### 2. 数据统计 (Stats)
- **大盘概览**：统计总记忆条数与会话总数。
- **可视化图表**：直观展示各类记忆的占比（饼图）、近 30 天记忆生成趋势（折线图）以及健康度（条形图）。
- **指标监控**：提供平均记忆强度、平均活跃度等核心质量指标。

### 3. 运行监控 (Monitor)
- **底层指标**：通过分析 SQLite 事件流，推断每小时活跃度与错误率。
- **事件流 (Event Stream)**：实时查看近期 AI 调用记忆工具的底层事件，掌握 AI 的“思考过程”。

### 4. 设置 (Settings)
- **可视化配置**：图形化修改 `config.jsonc` 中的核心参数（如最大记忆条数、注入模式、存储路径等）。
- **实时生效**：保存后立即生效，无需手动寻找隐藏配置文件。下方提供真实 JSONC 原文供比对。
- **重置功能**：支持一键恢复默认设置。

## 🚀 启动与使用说明

### 方式一：一键启动脚本（Windows 推荐）
在项目根目录双击运行 `start-viewer.bat` 即可自动编译并启动服务，随后会自动在浏览器中打开控制台。

### 方式二：命令行启动（日常使用）
适用于日常查看记忆状态，运行打包后的稳定服务端：

1. **首次使用或修改代码后需构建一次**：
   ```bash
   bun run build
   ```
2. **启动服务**：
   ```bash
   bun run viewer
   ```
3. **访问地址**：打开浏览器访问 **http://127.0.0.1:3456**

### 方式三：开发模式（二次开发使用）
适用于需要修改 Vue/Preact 前端代码并期望实时热更新（HMR）的开发者：

1. **终端 1 - 启动后端 API**：
   ```bash
   bun run viewer
   ```
2. **终端 2 - 启动前端 Vite 服务**：
   ```bash
   bun run viewer:dev
   ```
3. **访问地址**：打开浏览器访问 **http://127.0.0.1:5173**（Vite 会自动代理 API 请求至后端 3456 端口）。

## 🔒 安全说明
- Viewer 接口默认严格绑定在 **127.0.0.1**，拒绝任何外部网络访问。
- 所有敏感操作（保存设置、修改记忆）均具备**同源保护（Same-Origin Protection）**及自定义 Header 校验，有效防御 CSRF 攻击，防止恶意网页篡改您的记忆库。
